# adminApp

这是服装网站的后台管理系统，运行在本地浏览器，端口为3000，支持向cloudflare和wrangler本地模拟（数据库）环境上传（存储）、读取、更新、删除数据。

## 技术框架

- vite + react + tailwindcss
- UI库：shadcn/ui
- 数据库：cloudflare workers D1 R2 kv
- dnd-kit：用于组件排序

## 数据业务底层逻辑

我们采用了使用预定义组件，“积木式”页面编辑的方式：

1. adminApp需要实现复杂的页面数据管理业务，
2. 将数据存储至cloudflare数据库或本地模拟数据库，
3. 前端根据请求到的数据动态生成页面。

注意：最大的特点就是adminApp可以像“搭积木”一样自由编辑页面，前端则根据数据库中的数据动态生成页面。而不是硬编码或填入式页面内容。

## 数据说明

主要分为3种类型的数据：

1. 公司信息、header和footer。它们都是前端网页全局需要展示，不属于预定义组件。
2. 页面数据。会根据预定义组件的增、删、改、排序、隐藏等操作，动态生成页面数据，并存储至数据库。
3. 商品数据。

具备中英文内容管理，方便前端网站根据语言环境展示不同的内容。

### 2. 页面组成

后台管理页面分为左边的导航栏和右边的内容编辑区。
导航栏包括：公司信息管理、概览、header组件管理、footer组件管理、浏览组件、页面管理、商品管理、用户管理。

#### 1. 公司信息管理：

管理公司信息，包括公司名称、logo、联系方式、公司简介、地址、社交媒体链接等等，部分组件是不能编辑的，比如header组件中会展示公司名称和logo，footer组件中会展示公司名称和联系方式，这些内容是直接从公司信息数据中获取的。

#### 2. header组件管理：

管理header组件，包括导航栏等，注意链接只能从数据库中已有的页面中选择，如果页面被删除了，那么应该有（错误警告）提示：该页面已被删除，请更新。

#### 3. footer组件管理：

管理footer组件的内容，可以添加外部链接，注意：如果页面被删除了，那么应该有（错误警告）提示：该页面已被删除，请更新。

- 注意：输入链接时需要选择：内部/外部链接，内部链接只能进行选择，外部链接可以imput输入。

#### 4. 预定义组件管理：

浏览预定义组件的实际UI效果，包括轮播图、品牌价值、统计数据、评价、工厂板块等等。

#### 5. 页面管理：

展示目前所有的页面，可以添加、删除、编辑，点击目标可以进入编辑目标页面。
包括：固定页面和动态页面。

1. 动态页是可以删除的，
2. 固定页面是不能删除的，包括：首页、全部商品页、新品、工厂、FAQ。

##### 5.1 编辑页面：

管理各个页面的内容结构，包括：页面的链接地址、标题，可以选择任何预定义的组件（block），自由增删改、排序、隐藏页面组件等等，这样就可以实现自由对页面的内容进行编辑。

#### 6. 商品管理：

##### 6.1 商品编辑：

管理商品信息，包括商品名称、价格、图片、描述等等，可以添加、删除、编辑。

##### 6.2 商品分类管理：

管理商品分类，包括分类名称、图片、描述等等，可以添加、删除、编辑。
